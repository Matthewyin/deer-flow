# 业务基线监控技能

## 概述

此技能指导你完成XXX全业务统一运营支撑平台的每日运行数据分析。包括：解析每日报告文件、维护全历史平均基线、执行全量指标对比、查询历史趋势。

数据来源为 `docs/businessInfo/everybusiness` 文件（每日覆盖，只含前一天数据），解析后持久化到独立 SQLite 数据库 `business_baseline.db`。

## 触发条件

当用户输入包含以下任何一种情况时，立即加载此技能：

- 提到"业务基线"、"基线对比"、"业务报告"、"业务分析"
- 提到"在线终端"、"售票"、"兑奖"、"TPS"、"响应时间"
- 提到具体中心名（"支付中心"、"开放平台"、"营销中心"、"短信服务"、"合作渠道"等）
- 提到"趋势"、"历史数据"、"偏差"、"基线"
- 提到"业务数据"、"运行报告"、"运营报告"
- 需要查看某项指标是否正常

## 可用工具

| 工具 | 用途 | 数据源 |
|------|------|--------|
| `parse_daily_report` | 解析 everybusiness 文件，27项指标入库 | 文件 → SQLite |
| `query_current_data` | 查询最新一天完整指标数据 | SQLite (metrics) |
| `query_baseline` | 查询全历史平均基线 | SQLite (baseline) |
| `compare_with_baseline` | **核心工具**：最新数据 vs 基线全量对比 | SQLite (metrics + baseline) |
| `query_history_trend` | 查询指定指标历史趋势 | SQLite (metrics) |
| `get_interpretation` | 获取指标解读标准 | 内置常量 |

## 指标解读标准

> **重要**：以下标准仅作为参考范围，供 Agent 在输出时附加说明。Agent 不做异常判断，仅展示当前值与标准的对照关系。

### 终端类指标

| 指标 | 正常范围 | 关注阈值 | 危险阈值 | 说明 |
|------|---------|---------|---------|------|
| 在线终端数（总数） | >200,000台 | <180,000台 | <150,000台 | 反映终端在线率 |
| 平台售票峰值TPS | 1,500~5,000笔/秒 | <1,000 或 >6,000 | <500笔/秒 | 受开奖日影响显著 |

### 售票类指标

| 指标 | 正常范围 | 关注阈值 | 危险阈值 | 说明 |
|------|---------|---------|---------|------|
| 技术失败数 | 0~50次 | >100次 | >500次 | 系统稳定性核心指标 |
| 乐透售票峰值TPS | 1,000~5,000 | 偏差>50% | 偏差>80% | 开奖日（周二/四/日）TPS显著升高 |
| 竞彩售票峰值TPS | 1,000~2,000 | 偏差>50% | 偏差>80% | 相对稳定 |
| 传足/套票/套餐票TPS | 依票种不同 | 偏差>50% | 偏差>80% | 量级差异大，关注偏差率 |

### 兑奖类指标

| 指标 | 正常范围 | 关注阈值 | 危险阈值 | 说明 |
|------|---------|---------|---------|------|
| 技术失败数 | 0次 | >10次 | >50次 | 兑奖失败影响用户体验 |
| 业务失败数 | 0~500次 | >1,000次 | >5,000次 | 含正常业务拒绝 |

### 响应时间类指标

| 指标 | 正常范围 | 关注阈值 | 危险阈值 | 说明 |
|------|---------|---------|---------|------|
| 平均响应时间 | <200ms | >500ms | >1,000ms | 适用于乐透/传足/竞彩/套票/套餐票 |
| 最大响应时间 | <2,000ms | >5,000ms | >10,000ms | 关注是否有突增 |
| 中值响应时间 | <200ms | >500ms | >1,000ms | 更能反映典型体验 |

### 各中心指标

| 指标 | 正常范围 | 关注阈值 | 危险阈值 | 说明 |
|------|---------|---------|---------|------|
| 平均响应时间 | 依中心不同 | 偏差>100% | 偏差>200% | 量级差异大（0.8ms~300ms），关注偏差率 |
| 请求数 | 依中心不同 | 偏差>50% | 偏差>80% | 关注突变 |
| 请求失败数 | 0次 | >10次 | >50次 | 即开中心重点关注 |

### 量级参考

| 指标类型 | 典型量级 | 单位 |
|---------|---------|------|
| 在线终端数 | 230,000 | 台 |
| 乐透售票请求数 | 10,000,000~25,000,000 | 次 |
| 平台售票峰值TPS | 1,500~5,000 | 笔/秒 |
| 乐透响应时间(平均) | 70~100 | 毫秒 |
| 营销中心请求数 | 25,000,000~35,000,000 | 次 |
| 短信服务响应时间(平均) | 0.5~1.5 | 毫秒 |
| 合作渠道响应时间(平均) | 100~400 | 毫秒 |
| 合作渠道响应时间(最大) | 可达630,000+ | 毫秒 |

## 工作流程

### 流程 A：每日基线对比（默认流程）

示例输入：`"看看今天的业务基线"` / `"基线对比"` / `"业务报告分析"`

**步骤：**

1. **解析入库**：
   ```
   parse_daily_report()
   ```
   - 自动读取 `everybusiness` 文件
   - 解析27项指标，存入 SQLite
   - 自动更新全历史平均基线

2. **全量对比**：
   ```
   compare_with_baseline()
   ```
   - 获取最新日数据 vs 基线
   - 返回27项指标全量对比表（当前值、基线均值、偏差率）

3. **格式化输出**：
   - 输出全量指标对比表
   - 对偏差率超过50%的指标，附加解读标准（引用上表）
   - 标注报告日期和基线样本数

### 流程 B：单项指标趋势

示例输入：`"乐透售票TPS最近30天趋势"` / `"响应时间历史数据"`

**步骤：**

1. **查询历史趋势**：
   ```
   query_history_trend(metric_key="lotto_sell", days=30)
   ```

2. **输出时间序列 + 周期性分析**：
   - 展示时间序列数据
   - 标注开奖日（周二/四/日）和非开奖日的差异
   - 如有异常波动，附加解读标准

### 流程 C：指标解读查询

示例输入：`"支付中心的响应时间标准是什么？"` / `"技术失败多少算正常？"`

**步骤：**

1. **查询解读标准**：
   ```
   get_interpretation(metric_key="payment")
   ```

2. **输出解读标准**：
   - 展示该指标的正常范围、关注阈值、危险阈值
   - 提供量级参考

### 流程 D：指定日期数据查询

示例输入：`"昨天的业务数据"` / `"3月15日的售票数据"`

**步骤：**

1. **查询当前数据**：
   ```
   query_current_data()
   ```
   - 不传 metric_key 返回全部27项指标

2. **格式化输出**：按类别分组展示指标数据

### 流程 E：模糊场景澄清

当用户描述模糊时（如"业务怎么样"），主动建议：
- "要查看完整的基线对比报告吗？"
- "要查看某项具体指标的趋势吗？"
- "要查看某类指标（售票/兑奖/响应时间/各中心）的数据吗？"

## 语义理解要点

- "看看业务基线" → parse_daily_report() + compare_with_baseline()
- "今天的数据正常吗" → parse_daily_report() + compare_with_baseline() + get_interpretation()
- "乐透TPS趋势" → query_history_trend(metric_key="lotto_sell", days=30)
- "支付中心响应时间" → query_current_data(metric_key="payment")
- "终端数历史" → query_history_trend(metric_key="online_terminals", days=30)
- "技术失败多少算正常" → get_interpretation()
- "昨天兑奖数据" → query_current_data()
- "合作渠道响应时间标准" → get_interpretation(metric_key="partner")

## 输出规范

### 基线对比报告
- 全量指标表：27项全部展示（指标名、当前值、基线均值、偏差率）
- 偏差率格式：`+15.3%` 或 `-8.2%`，保留1位小数
- 对偏差率>50%的指标加粗标注并附加解读标准

### 趋势报告
- 时间序列表格（日期、值）
- 标注开奖日/非开奖日差异
- 提供周期性分析

### 未匹配到结果时
- 明确告知："数据库中暂无对应数据，请先执行 parse_daily_report() 解析数据文件"

## 注意事项

- 文件每天被覆盖，必须及时调用 parse_daily_report() 持久化数据
- 基线计算方式为全历史平均，非滑动窗口
- 偏差率 = (当前值 - 基线均值) / 基线均值 × 100%
- 基线均值为0时，偏差率不计算（显示 N/A）
- 数值中的逗号为千分位分隔符（如"1,041,555"），解析时需去除
- 开奖日（周二/四/日）乐透TPS显著高于非开奖日，趋势分析时需标注
- 系统与带宽管理、知识库、远程探针系统完全独立，无数据交互
