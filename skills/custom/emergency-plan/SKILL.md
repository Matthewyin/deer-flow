# 运维知识库技能

## 概述

此技能指导你完成运维知识库的完整交互流程：从理解用户需求，到检索知识、上传文档、浏览库内容。

知识库存储在独立的 ChromaDB 向量数据库中，通过 MCP Server 提供服务。支持以下五种文档类型：

| doc_type | 说明 | 典型内容 |
|----------|------|---------|
| `emergency` | 应急预案 | 故障应急处置流程、操作步骤 |
| `sop` | 标准操作流程 | 日常运维操作规范 |
| `fault` | 故障案例 | 历史故障排查过程与结论 |
| `solution` | 解决方案 | 问题解决方案与技术方案 |
| `event` | 事件记录 | 运维事件报告与记录 |

## 触发条件

当用户输入包含以下任何一种情况时，立即加载此技能：
- 提到"应急预案"、"预案"、"应急响应"、"应急处置"
- 提到"故障处置"、"故障处理步骤"、"紧急处理"
- 提到"P1"、"P2"、"重大故障"、"紧急故障"
- 提到"SOP"、"标准操作"、"操作流程"、"操作规范"
- 提到"故障案例"、"排障案例"、"troubleshooting"
- 提到"解决方案"、"知识库"、"运维文档"
- 提到"事件记录"、"事件报告"
- 描述了具体的故障场景并需要操作指导
- 需要查找历史运维经验或文档
- 需要上传或管理运维文档

## 可用工具

| 工具 | 用途 | 数据源 |
|------|------|--------|
| `knowledge_search` | 语义检索知识库 | ChromaDB (ops_knowledge collection) |
| `knowledge_upload` | 文档入库（支持pdf/docx/xlsx/txt/md/csv） | ChromaDB + SQLite |
| `knowledge_list` | 浏览知识库文档列表 | SQLite (metadata.db) |

## 工作流程

### 流程 A：故障场景查询（应急预案）

示例输入：`"核心交换机宕机怎么处理？"`

**步骤：**

1. **理解故障场景**：从用户描述中提取关键信息
   - 故障设备/系统、故障现象、优先级

2. **检索应急预案**：
   ```
   knowledge_search(query="核心交换机宕机应急处置", doc_type="emergency", top_k=5)
   ```

3. **整合与输出**：将检索结果按以下结构整理：
   ```markdown
   ## 故障场景：[用户描述的故障场景]
   ### 匹配预案：[预案名称/文档ID]
   ### 操作步骤：
   1. [步骤1]：[操作内容] — [预期结果]
   2. [步骤2]：[操作内容] — [预期结果]
   ...
   ### 注意事项：
   - [注意事项1]
   ### 依据来源：[知识库文档引用]
   ```

### 流程 B：知识检索（SOP/故障案例/解决方案/事件）

示例输入：`"华为交换机OSPF邻居关系断开的排查方法"`

**步骤：**

1. **理解用户意图**：识别为知识检索需求
   - 关键词：OSPF、邻居断开、华为交换机
   - 隐含过滤：device_vendor=huawei, device_type=switch

2. **调用知识检索**：
   ```
   knowledge_search(query="华为交换机OSPF邻居关系断开排查", device_vendor="huawei", top_k=5)
   ```

3. **整合回复**：将检索结果按相关性整理后回复用户

### 流程 C：文档入库

示例输入：`"把这个SOP文档加到知识库里"`

**步骤：**

1. **确认文档信息**：
   - 文件路径（用户指定或从上下文获取）
   - 文档类型：sop / fault / emergency / solution / event
   - 标题
   - 可选：设备厂商、设备类型

2. **调用入库工具**：
   ```
   knowledge_upload(
       file_path="docs/ops-knowledge/raw/SOP/02网络SOP/OSPF配置规范.docx",
       doc_type="sop",
       title="OSPF配置规范",
       device_vendor="huawei"
   )
   ```

3. **确认结果**：告知用户入库状态和chunk数量

### 流程 D：浏览知识库

示例输入：`"知识库里有哪些故障案例？"`

**步骤：**

1. **调用浏览工具**：
   ```
   knowledge_list(doc_type="fault", limit=20)
   ```

2. **展示结果**：列出文档标题、类型、上传日期

### 流程 E：模糊场景澄清

当用户描述模糊时（如"网络出问题了"），先要求用户补充：
- 具体故障现象（断网、丢包、延迟高、设备宕机等）
- 影响范围（单台设备、单机房、全区域）
- 故障等级（P1/P2/P3）
- 涉及设备厂商和型号（如有）

### 流程 F：混合查询

用户可能在检索后要求上传新文档，或在浏览后进行检索。按需组合上述流程。

## 语义理解要点

- "网络断了" → knowledge_search(query="网络中断应急预案", doc_type="emergency")
- "交换机挂了" → knowledge_search(query="交换机故障宕机应急", doc_type="emergency")
- "怎么处理XX故障" → knowledge_search(query="XX故障处理")
- "有没有XX的SOP" → knowledge_search(query="XX操作流程", doc_type="sop")
- "把这个文件加进去" → knowledge_upload(...)
- "库里有什么" → knowledge_list()
- "华为相关的故障案例" → knowledge_search(query="故障案例", device_vendor="huawei")
- "P1故障怎么处理" → knowledge_search(query="P1重大故障应急响应", doc_type="emergency")
- "有哪些预案" → knowledge_list(doc_type="emergency")

## 输出规范

### 应急预案匹配时
- 严格按照预案原文输出操作步骤，不做主观修改
- 所有内容标注知识库来源
- 如有多条匹配预案，列出供用户选择

### 未匹配到结果时
- 明确告知："当前知识库中未查询到对应内容"
- 建议用户补充细节或联系运维专家
- **绝不编造不存在的预案或操作步骤**

### 多结果匹配时
- 列出所有匹配结果的名称和简要描述
- 由用户选择后再展开详细内容

## 注意事项

- 文档入库时自动去重：同hash跳过，同标题覆盖更新
- 支持6种文件格式：pdf、docx、xlsx、txt、md、csv
- 检索支持元数据过滤：doc_type、device_vendor、device_type
- 知识库与带宽管理系统相互隔离，使用独立的ChromaDB collection和SQLite数据库
- **来源可追溯**：所有输出必须标注知识库来源文档
- **不编造**：知识库中无对应内容时，明确告知，不虚构操作步骤
