# 应急预案技能

## 概述

此技能指导你完成应急预案查询与分析的完整流程：从理解用户的故障场景描述，到检索知识库中的应急预案，输出结构化的故障处置操作步骤。

**重要：本技能仅覆盖应急预案（emergency）类型的文档查询。** SOP、故障案例、解决方案等其他运维知识由 `ops-knowledge` 技能负责。

## 触发条件

当用户输入包含以下任何一种情况时，立即加载此技能：
- 提到"应急预案"、"预案"、"应急响应"、"应急处置"
- 提到"故障处置"、"故障处理步骤"、"紧急处理"
- 提到故障等级如"P1"、"P2"、"重大故障"、"紧急故障"
- 描述了具体的故障场景并需要操作指导
- 需要查看故障应急处置流程

## 可用工具

| 工具 | 用途 | 数据源 |
|------|------|--------|
| `knowledge_search` | 语义检索应急预案 | ChromaDB (ops_knowledge collection, doc_type=emergency) |
| `knowledge_list` | 浏览应急预案文档列表 | SQLite (metadata.db, doc_type=emergency) |

**注意：** 不使用 `knowledge_upload` 工具（应急预案入库由管理员统一处理，不在agent交互流程中）。

## 工作流程

### 流程 A：故障场景查询

示例输入：`"核心交换机宕机怎么处理？"`

**步骤：**

1. **理解故障场景**：从用户描述中提取关键信息
   - 故障设备/系统：核心交换机
   - 故障现象：宕机
   - 优先级：隐含P1（核心设备故障）

2. **检索应急预案**：
   ```
   knowledge_search(query="核心交换机宕机应急处置", doc_type="emergency", top_k=5)
   ```

3. **整合与输出**：将检索结果按以下结构整理：
   ```markdown
   ## 故障场景：[用户描述的故障场景]
   ### 匹配预案：[预案名称/文档ID]
   ### 操作步骤：
   1. [步骤1]：[操作内容] — [预期结果]
   2. [步骤2]：[操作内容] — [预期结果]
   ...
   ### 注意事项：
   - [注意事项1]
   - [注意事项2]
   ### 依据来源：[知识库文档引用]
   ```

### 流程 B：预案浏览

示例输入：`"有哪些网络相关的应急预案？"`

**步骤：**
1. 调用浏览工具：
   ```
   knowledge_list(doc_type="emergency", limit=50)
   ```
2. 按设备类型/故障类型分类展示给用户

### 流程 C：模糊场景澄清

当用户描述模糊时（如"网络出问题了"），先要求用户补充：
- 具体故障现象（断网、丢包、延迟高、设备宕机等）
- 影响范围（单台设备、单机房、全区域）
- 故障等级（P1/P2/P3）
- 涉及设备厂商和型号（如有）

## 语义理解要点

- "网络断了" → knowledge_search(query="网络中断应急预案", doc_type="emergency")
- "交换机挂了" → knowledge_search(query="交换机故障宕机应急", doc_type="emergency")
- "核心路由器异常" → knowledge_search(query="核心路由器故障应急", doc_type="emergency")
- "防火墙策略导致业务中断" → knowledge_search(query="防火墙策略故障业务中断应急", doc_type="emergency")
- "有哪些预案" → knowledge_list(doc_type="emergency")
- "P1故障怎么处理" → knowledge_search(query="P1重大故障应急响应", doc_type="emergency")

## 输出规范

### 匹配到预案时
- 严格按照预案原文输出操作步骤，不做主观修改
- 所有内容标注知识库来源
- 如有多条匹配预案，列出供用户选择

### 未匹配到预案时
- 明确告知："当前知识库中未查询到对应故障场景的应急预案"
- 建议用户补充故障细节或联系运维专家
- **绝不编造不存在的应急预案内容**

### 多预案匹配时
- 列出所有匹配预案的名称和简要描述
- 由用户选择后再展开详细步骤

## 注意事项

- **仅查询应急预案**：不处理SOP查询、带宽分析、配置管理等其他运维任务
- **只读不写**：只检索知识库，不上传/修改/删除文档
- **来源可追溯**：所有输出必须标注知识库来源文档
- **不编造**：知识库中无对应内容时，明确告知，不虚构操作步骤
